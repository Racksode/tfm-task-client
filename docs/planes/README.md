# Planes revisados con IA

> Carpeta interna de trazabilidad metodológica del TFM.

Esta carpeta registra planes importantes propuestos por agentes IA, revisados por el usuario y utilizados como apoyo metodológico durante la definición del MVP.

Su objetivo no es duplicar la documentación principal del proyecto, sino dejar evidencia breve de:

- qué bloque documental o técnico se planificó
- qué ficheros se consideraron afectados
- qué ajustes humanos se aplicaron antes de ejecutar
- qué resultado quedó incorporado al repositorio
- qué commit se asoció o se propuso para el cambio

Los documentos principales del proyecto siguen siendo la fuente de verdad funcional y técnica. Los planes de esta carpeta actúan como trazabilidad complementaria para explicar el uso controlado de IA durante el TFM.

## Criterio de uso

Solo deben registrarse planes relevantes para fases, decisiones o bloques de trabajo importantes.

No se deben guardar conversaciones menores, propuestas descartadas sin impacto o documentación extensa que repita el contenido de `docs/`.
