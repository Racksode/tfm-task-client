# Uso de inteligencia artificial en el proyecto

**Documento interno de trabajo**  
**Idioma:** Español  
**Tipo de documento:** Definición del uso de IA en el TFM  

---

## 1. Objetivo

Este documento define el papel de la inteligencia artificial dentro del proyecto.

El objetivo no es que la IA controle toda la plataforma, sino incorporarla en funcionalidades concretas, útiles y demostrables dentro del MVP del TFM.

La IA debe aportar valor real al flujo de trabajo: reducir fricción, mejorar la calidad de los reportes, transformar información técnica en lenguaje comprensible para el cliente y abrir la puerta a futuras integraciones con agentes externos.

---

## 2. Funcionalidades IA incluidas en el TFM

Para mantener un alcance realista, el TFM debería centrarse en una o dos funcionalidades IA bien integradas.

### 2.1. Generación de resúmenes profesionales

A partir de tareas y registros de tiempo, la IA podrá generar un resumen claro y profesional del trabajo realizado durante un periodo.

Ejemplo de notas internas:

> Ajuste de endpoint, revisión de errores 500, modificación de validación y pruebas con datos reales.

Ejemplo de texto orientado al cliente:

> Durante este periodo se han realizado ajustes técnicos en el sistema para mejorar la estabilidad de algunos procesos, corregir errores detectados y validar el funcionamiento con datos reales.

### 2.2. Transformación de lenguaje técnico

La IA podrá convertir descripciones técnicas internas en textos más comprensibles para el cliente.

Esto permitirá mantener notas útiles para el equipo interno y, al mismo tiempo, generar comunicaciones más profesionales y entendibles para el área de cliente o los reportes.

### 2.3. Prueba conceptual de instrucciones en lenguaje natural

Como demostración adicional, el MVP podría incluir una pantalla interna llamada, por ejemplo, `Asistente IA`, donde el usuario escriba una instrucción en lenguaje natural.

Ejemplo:

> Registra 1 hora y 30 minutos en la tarea "Revisión formulario contacto" del cliente Virtual Enjoy.

La IA interpretaría la instrucción y devolvería una propuesta estructurada:

```json
{
  "accion": "registrar_tiempo",
  "cliente": "Virtual Enjoy",
  "tarea": "Revisión formulario contacto",
  "duracion_minutos": 90,
  "requiere_confirmacion": true
}
```

La acción no debería ejecutarse automáticamente sin confirmación del usuario.

Este enfoque permite demostrar el concepto de agentes o asistentes sin asumir todavía la complejidad de integraciones reales con WhatsApp, email u otros canales externos.

---

## 3. Funcionalidades IA previstas como evolución posterior

Tras el TFM, la IA podría ampliarse con:

- Clasificación automática de tareas.
- Sugerencia de prioridades.
- Detección de tareas mal descritas.
- Generación automática de reportes mensuales.
- Comparación entre tiempo estimado y tiempo real.
- Detección de desviaciones.
- Asistente conversacional para consultar el estado de proyectos.
- Generación de comunicaciones para clientes.
- Recomendaciones sobre facturación.
- Análisis de rentabilidad por cliente o proyecto.

---

## 4. Integración futura con agentes IA

La plataforma podría evolucionar hacia la integración con agentes IA externos, como OpenClaw, Hermes u otros asistentes personalizados.

Estos agentes podrían recibir información desde canales como WhatsApp, correo electrónico o formularios conversacionales, interpretar el mensaje y convertirlo en acciones dentro del SaaS.

Ejemplos:

- Crear una tarea a partir de un mensaje.
- Registrar tiempo trabajado.
- Actualizar el estado de una tarea.
- Consultar horas acumuladas por cliente.
- Generar un resumen semanal.
- Preparar un reporte para enviar al cliente.

Esta integración se considera una línea futura, no parte del alcance principal del TFM.

---

## 5. Criterios de seguridad y control

Cualquier funcionalidad IA que proponga acciones dentro del sistema deberá contemplar:

- Confirmación explícita del usuario antes de ejecutar acciones relevantes.
- Validación de permisos.
- Registro de auditoría.
- Manejo de ambigüedades.
- Prevención de instrucciones maliciosas o incorrectas.
- Posibilidad de revisar la información generada antes de enviarla al cliente.

La IA debe actuar como asistente, no como sustituto completo del criterio del usuario.
